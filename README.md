# 언리얼 멀티플레이어 숫자 야구 게임
<img width="2559" height="751" alt="image" src="https://github.com/user-attachments/assets/076b6ecc-d7f3-4f0b-a7f0-4246dbc3e681" />

## 소개
본 프로젝트는 언리얼 엔진의 서버-클라이언트 구조와 데이터 동기화 흐름을 이해하기 위해 제작된 멀티플레이어 숫자 야구 게임입니다. 서버가 공정하게 게임 규칙을 주관하고, 클라이언트는 그 결과를 받아 화면에 시각적으로 출력하는 온라인 게임 개발의 핵심 프로세스를 경험하는 것을 목표로 합니다

## 학습 목표
- **멀티플레이어 기초:** Server/Client 간의 역할 분담과 RPC 통신 이해
- **PlayerState :** 플레이어의 개별 데이터(시도 횟수 등)를 독립적으로 관리/동기화하기
- **GameModeBase:** 게임의 전체적인 규칙(판정, 승리 조건)을 서버 측에서 제어하기
- **UMG 연동:** C++ 로직의 결과를 위젯을 통해 플레이어에게 시각적으로 전달하기
- **유효성 검사:** 잘못된 입력이 서버 로직을 해치지 않도록 방어 코드 작성하기

## 게임 규칙
- 목표: 서버가 숨긴 1~9 사이의 중복되지 않는 3자리 비밀 숫자를 맞히는 게임입니다.
- 판정 시스템:
  - S (스트라이크): 숫자와 자리가 모두 일치할 때. 
  - B (볼): 숫자는 맞지만 자리가 다를 때.
  - OUT (아웃): 입력한 숫자가 정답에 하나도 없을 때.
- 승리/종료 조건: * 각 플레이어에게는 단 3번의 기회가 주어집니다.
  - 승리: 3번의 기회 안에 먼저 3S를 달성하는 플레이어가 승리합니다.
  - 무승부: 모든 플레이어가 기회를 소진해도 정답자가 없으면 무승부 처리됩니다.
- 턴제 규칙:
  - 턴 제한 시간: 각 턴마다 주어진 시간 내에 숫자를 입력해야 합니다.
  - 시간 초과: 시간 내에 입력하지 못하면 **기회가 1회 소진**되며 다음 사람에게 차례가 넘어갑니다.

## 요구사항 분석 및 설계
### 필수 구현 기능
1. **GameModeBase의 판정 로직 구현**
    - [x]  기본적인 멀티플레이 채팅 구현
    - [x]  정답 숫자 3자리를 생성하는 함수 구현
        - 게임 시작 시 서버에서 1~9 사이의 중복되지 않는 3자리 숫자 생성
        - 예: 386, 594
    - [x]  숫자 입력 검사 로직 구현: 사용자가 입력한 숫자가 규칙에 맞는지 검사합니다.
        - 3자리 숫자인가?
        - 문자가 포함되지는 않았는가?
        - 중복되는 숫자가 있는가?
        - 위 조건에 맞지 않으면 "다시 입력하세요"라는 안내를 출력하고 기회를 소진하지 않습니다.
    - [x]  판정 로직 구현: 제시된 숫자와 정답을 비교하여 결과를 계산합니다.
        - **S(Strike):** 숫자와 자리가 모두 일치
        - **B(Ball):** 숫자는 존재하지만 자리가 불일치
        - **OUT:** 일치하는 숫자가 하나도 없음
        - 결과값은 "1S1B", "OUT" 등의 문자열로 반환합니다.
    
2. **PlayerState 클래스를 활용한 시도 횟수 관리**
    - [x]  기회 제한 시스템 구현
        - 각 플레이어에게 기본 3회의 기회를 부여합니다.
    - [x]  시도 횟수 업데이트 구현
        - 각 플레이어들의 현재 시도 횟수를 관리합니다.
        - 채팅(숫자 입력)을 입력할 때마다 현재 시도 횟수를 1씩 증가시킵니다.
    - [x]  플레이어가 숫자를 입력할 때마다 현재 시도 횟수와 최대 시도 횟수가 보이게 구현합니다.
        - Getter 함수 구현
        - 예 ) `[현재 시도 횟수 / 최대 시도 횟수]`
    

3. **게임 제어 : 승리, 무승부 그리고 리셋**
    - [x]  게임 결과(승리, 무승부)를 알리기 위한 공지 위젯 생성.
    - [x]  적절한 클래스에서 현재 게임 결과를 판단하는 멤버 함수 구현.
        - [x]  승리 판정
            - 누구든 3번의 기회 안에 3S를 먼저 달성하면 즉시 해당 플레이어를 Winner로 판정합니다.
            - 승리한 플레이어가 누구인지 공지 위젯을 알립니다.
        - [x]  무승부 판정
            - 모든 플레이어가 3번의 기회를 모두 소모했음에도 3S를 맞추지 못했다면 무승부로 판정합니다.
            - 무승부 사실을 공지 위젯으로 알립니다.
    - [x]  게임 리셋 시스템
        - [x]  게임 결과 공지 위젯을 띄운 후(Winner가 나오거나 무승부), 
        모든 데이터를 초기화하고 정답 숫자를 새로 생성하여 게임을 재시작합니다.
          
### 도전 기능
1. **턴 제어 기능**
    - [x]  **실시간 타이머 위젯:** 화면에 남은 시간이 줄어드는 UI를 띄웁니다.
    - [x]  **시간 동기화**
        - 서버에서 남은 시간을 관리합니다.
        - 위젯에 그 값을 바인딩하여 모든 플레이어가 동일한 시간을 보게 합니다.
    - [x]  **입력 차단**
        - 남은 시간이 0초가 되면 숫자 야구 플레이를 할 수 없습니다.
        - 입력창을 막거나, 숫자를 입력해도 서버에서 받아주지 않도록 구현해보세요.
    

2. **턴 내에 숫자 야구(채팅을 입력)하지 않을 시 기회 소진 기능**
    - [x]  **활동 체크:** 해당 턴에 숫자를 입력했는지 여부를 판별하는 체크 변수를 활용합니다.
    - [x]  **턴 전환 시스템 :** 플레이어가 숫자를 입력하거나 시간이 종료될 경우
        - [x]  다음 플레이어에게 플레이 권한을 넘깁니다.
        - [x]  시도 횟수를 1회 차감합니다.
    
### 서버-클라이언트 데이터 흐름 설계
1. 게임 시작 및 초기화 (Server 단독)
    - 주체: `GameModeBase` (서버에만 존재)
    - 동작: 게임이 시작되면 서버의 `GameModeBase`에서 1~9 사이의 중복되지 않는 3자리 정답 숫자를 생성하여 서버 메모리에 안전하게 보관합니다. 클라이언트는 이 정답을 절대 알 수 없습니다.

2. 플레이어 입력 전달 (Client → Server)
    - 주체: `PlayerController` (클라이언트에서 입력받아 서버로 전달)
    - 동작: 클라이언트(플레이어)가 채팅창에 3자리 숫자를 입력합니다.
    - 통신 방식: `Server` 지정자가 붙은 RPC(Remote Procedure Call) 함수를 호출하여, 클라이언트의 입력값을 서버로 전송합니다.

3. 유효성 검사 및 판정 (Server 단독)
    - 주체: `GameModeBase`
    - 동작: 
        - 유효성 검사: 서버가 전달받은 데이터가 규칙에 맞는지(3자리인지, 문자/중복은 없는지) 검사합니다. 규칙에 어긋나면 기회 차감 없이 클라이언트에게 "다시 입력하세요" 라고 응답합니다.
        - 결과 판정: 올바른 입력이라면, 서버가 보관 중인 정답과 비교하여 S(스트라이크), B(볼), OUT 판정을 계산합니다.

4. 상태 업데이트 및 동기화 (Server → Client)
    - 주체: `PlayerState` (서버에서 생성되어 모든 클라이언트로 복제됨)
    - 동작: 유효한 입력을 마친 플레이어의 현재 시도 횟수를 서버에서 1 증가시킵니다.
    - 통신 방식: `PlayerState`의 변수 값이 변경되면, 언리얼 엔진의 프로퍼티 리플리케이션을 통해 변경된 시도 횟수 데이터가 자동으로 모든 클라이언트에 복제 및 동기화되어 화면(UMG)에 표시됩니다.

5. 결과 브로드캐스팅 및 게임 제어 (Server → All Clients)
    - 주체: `GameModeBase` → 클라이언트 UMG
    - 동작: 판정 결과 ("1S1B" 등)나 승리/무승부 여부를 모든 플레이어에게 알립니다. 승리/무승부 판정이 났을 경우 서버 데이터를 초기화하고 정답을 새로 생성합니다.
    - 통신 방식: `NetMulticast` 지정자가 붙은 RPC 함수를 호출하여, 서버가 접속한 모든 클라이언트에게 공지 위젯을 띄우도록 명령을 내립니다.

6. 턴 제어와 타이머
    - 서버가 남은 시간을 관리하며 1초마다 감소시킵니다.
    - 이 남은 시간 데이터를 리플리케이션을 통해 클라이언트의 위젯에 바인딩하여 모두가 동일한 타이머를 보게 합니다.
    - 시간이 0초가 되면 서버 측에서 해당 플레이어의 입력을 차단하고, 시도 횟수를 차감한 뒤 다음 플레이어로 턴(권한)을 넘깁니다.
    
## 구현
### 1. 코어 게임 로직 (서버의 영역)
가장 깊은 곳에는 게임의 규칙을 통제하고 정답을 알고 있는 절대적인 심판이 존재합니다. 이 역할은 오직 서버에만 존재하는 **`NBGameModeBase`**가 담당합니다.

- **정답 생성 및 검증:** 게임이 시작되면 1~9 사이의 중복 없는 3자리 숫자(정답)를 생성합니다. 클라이언트가 숫자를 보내면, `IsValidInput`으로 규칙에 맞는지 검사하고 `CheckAnswer`를 통해 스트라이크와 볼을 판정합니다.
- **턴 관리 (`StartNextTurn`, `OnTurnTimerTick`):** 누구의 턴인지 결정하고 10초의 타이머를 굴립니다. 시간이 초과되면 강제로 기회를 1회 차감하고 다음 사람에게 턴을 넘깁니다.
- **전체 공지 (`Multicast_BroadcastResult`):** 승리나 무승부 같은 게임의 중대한 결과가 나왔을 때, 접속한 모든 플레이어의 화면에 팝업을 띄우도록 전체 방송을 송출합니다.

### 2. 상태 동기화 (전광판과 신분증)
멀티플레이 게임이므로, "지금 누구 차례인가?"와 "저 사람은 기회를 몇 번 썼는가?"를 모두가 똑같이 알아야 합니다. 이 데이터를 복제(Replication)하여 모두에게 공유하는 시스템입니다. 

- **공용 전광판 (`NBGameStateBase`):** 현재 턴인 플레이어(`CurrentTurnPlayer`)와 남은 시간(`TurnTimeRemaining`)을 가지고 있습니다. 이 정보는 서버가 업데이트하면 모든 클라이언트의 화면에 동일하게 반영됩니다.
- **개인 신분증 (`NBPlayerState`):** 각 플레이어의 현재 시도 횟수(`CurrentAttempt`)를 기록합니다. 클라이언트가 임의로 조작할 수 없도록, 숫자를 올리는 작업(`IncrementAttempt`)은 오직 서버 권한(`HasAuthority`)으로만 실행됩니다.

### 3. 통신과 명령 전달 (컨트롤러의 영역)
클라이언트(플레이어의 컴퓨터)와 서버는 직접 데이터를 만질 수 없으므로, **`NBPlayerController`**가 그 사이에서 무전기(RPC) 역할을 수행합니다.

- **입력 전달 (Server RPC):** 플레이어가 UI에 숫자를 적고 엔터를 치면, 컨트롤러가 `ServerRPCSubmitNumberGuess`를 통해 서버로 "나 이 숫자 제출할게!"라고 보냅니다.
- **결과 수신 (Client RPC):** 서버가 판정을 마치면, 컨트롤러의 `ClientRPCAddGuessHistory`나 `ClientRPCShowAnnouncement`를 호출하여 "너의 화면(UI)에 1S1B라고 띄워라!"라고 명령을 내립니다.

### 4. 모듈형 UI 아키텍처 (클라이언트의 영역)
하나의 거대한 UI가 모든 것을 처리하지 않고, 매니저와 실무자들로 역할을 쪼갰습니다.

- **UI 총괄 매니저 (`NBHUDWidget`):**
    - 매 프레임(`NativeTick`)마다 전광판(`GameState`)과 내 신분증(`PlayerState`)을 확인합니다.
    - 데이터를 읽어온 뒤, 자신이 거느린 하위 위젯들에게 "시간 업데이트해!", "내 턴이니까 입력창 켜!"라고 지시를 내립니다. 스스로 텍스트를 바꾸지 않습니다.
- **독립된 UI 모듈들 (실무자):**
    - **`NBStatusBarWidget`:** 매니저가 던져주는 숫자와 이름만 받아서 상단 상태바에 예쁘게 포맷팅하여 그려줍니다.
    - **`NBInputWidget`:** 하단에서 키보드 입력을 기다리다가, 엔터를 치면 컨트롤러에게 문자열을 넘겨주고 스스로 텍스트 박스를 비웁니다.
    - **`NBHistoryWidget`:** 판정 결과가 내려오면, 새로운 텍스트 블록을 생성해 스크롤 박스 맨 밑에 추가하고 자동으로 스크롤을 내려줍니다.
    - **`NBAnnouncementWidget`:** 평소에는 숨어있다가, 승리/무승부 시 중앙에 나타나 정해진 시간 동안 팝업을 띄운 뒤 스스로 사라집니다.

### 전체 데이터 흐름

이 모든 것이 결합되어 하나의 턴이 진행되는 과정은 다음과 같습니다.

1.  **[상황]** 내 턴이 되어 **입력창(`NBInputWidget`)**이 활성화됩니다.
2.  **[입력]** "123"을 치고 엔터를 누르면, **컨트롤러(`NBPlayerController`)**가 서버로 통신(Server RPC)을 보냅니다.
3.  **[판정]** **서버(`NBGameModeBase`)**가 정답을 확인하고 "1S1B"라는 결과를 도출한 뒤, 내 시도 횟수를 1 증가시킵니다.
4.  **[결과 전달]** 서버가 다시 **컨트롤러**에게 통신(Client RPC)을 보내 "결과를 UI에 그려라"라고 명령합니다.
5.  **[시각화]** 컨트롤러가 **HUD 매니저(`NBHUDWidget`)**를 통해 **기록창(`NBHistoryWidget`)**에 "123 ➔ 1S1B"를 한 줄 추가합니다.
6.  **[턴 종료]** 서버는 턴을 다음 사람으로 넘기고, 다시 10초 타이머를 굴리기 시작합니다.
